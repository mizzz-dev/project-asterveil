# RELEASE READINESS ASSESSMENT

本書は、公開リポジトリの現状実装（2026-04-29時点）を基準に、Vertical Slice から製品リリースへ進むための不足を可視化する棚卸し資料である。

## 1. 現在の到達点（公開実装ベース）

- `game/` 配下に battle / quest / save / app / shop / location を中心とした責務分離が成立している。
- `data/master` と `data/save_contract` によるデータ駆動の骨格がある。
- `tests/` に `unittest` が整備され、Vertical Slice 単位の回帰確認が可能。
- 直近PR #54（工房ランク3解放後の特別依頼チェーン）までで、工房導線は「ランク解放→依頼連鎖→報酬」の最小ループが追加済み。

## 2. 既存 Vertical Slice で成立しているもの

- コマンド戦闘の最小導線（行動選択、進行、結果反映）
- クエスト進行の状態遷移（受注/進行/完了）
- セーブ契約に沿った保存/復元の最小導線
- Playable Slice としての「探索/戦闘/クエスト/工房」連携
- 工房系拡張（装備強化、分解、上位クラフト、ランク3連鎖依頼）

## 3. 製品化に不足しているもの（カテゴリ別）

### 3.1 クライアント / UI
- CLI中心の導線を、一般ユーザー向けUI（タイトル、設定、導線固定、終了復帰）へ移行する実装が不足。
- パッド前提操作、画面遷移、アクセシビリティ（フォントサイズ/色覚配慮）の定義が不足。

### 3.2 セーブ互換 / データ移行
- `save_data_v1` の将来互換ポリシーはあるが、移行手順（v1→v2）と自動検証フローが未整備。
- リリース後パッチを想定したセーブ移行テストマトリクスが不足。

### 3.3 本番用コンテンツ量
- MVP計画上の3章/サブ20件に対し、公開実装は「縦切り成立」を優先した最小量で、量産体制の検証が未完。
- 長編シナリオとエンドコンテンツは仕様はあるが、本番投入量の確定が不足。

### 3.4 QA / デバッグ
- 単体/結合テストはあるが、実プレイ観点の回帰（チュートリアル、入力切替、長時間プレイ）が不足。
- デバッグCLI依存機能の本番向け置換方針（隠しメニュー/開発フラグ分離）が未整理。

### 3.5 パフォーマンス
- Steam向け最低スペック・モバイル中位端末の目標値（fps、ロード時間、メモリ）を満たす検証計画が不足。

### 3.6 配布 / ビルド
- ストア提出可能形式のクライアントビルド手順（CI/CD、署名、配布パッケージ）が未定義。
- Python実装から提出用クライアントへ接続するアーキテクチャ判断（ラッパー/移植方針）が不足。

### 3.7 ストア要件
- Steamストア素材、説明文、年齢レーティング、法務表記の準備タスクが未分解。
- App Store / Google Play の審査導線（課金・個人情報・クラッシュ報告）が未着手。

### 3.8 運用 / ログ / クラッシュ収集
- テレメトリ方針はあるが、運用用ログ設計、クラッシュ収集、障害一次切り分け手順が不足。

### 3.9 マルチプレイ / フレンド機能
- 企画上は重要だが、現状は基礎仕様段階。初回リリース必須にすると全体遅延リスクが高い。

## 4. 現状評価（ステージ判定）

- **現状は「Internal Playable」相当**と評価する。
  - 理由: Vertical Slice の複数導線は成立している一方、ストア提出に必要なUI/QA/配布運用が未整備。
- **Demo-ready** 到達には、UI最小化・配布手順・QAチェックリスト整備が追加で必要。
- **Early Access-ready** には、セーブ互換運用とクラッシュ収集、コンテンツ供給計画の確立が必要。
- **1.0-ready** は、長編本編量・品質保証・運用体制を含むため現時点では対象外。

## 5. 直近PR #54を踏まえた差分認識

- PR #54 で工房ランク3解放後の特別依頼チェーンが追加され、コンテンツ連鎖の手触りは前進した。
- ただし製品化観点では、追加されたのは「ゲーム内ループの厚み」であり、ストア提出準備（UI・配布・QA・運用）は依然ギャップが大きい。
- よって今回の棚卸しでは、機能追加よりも「リリース可能性を決める不足の明文化」を優先する。
