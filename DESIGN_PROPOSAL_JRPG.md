# 新規長編コマンドRPG企画書（実務提案）

## 1. エグゼクティブサマリー

### 一言コンセプト
**「失われた記憶を編み直し、世界と仲間の“明日”を再定義する、群像長編コマンドRPG」**

### 想定タイトル案（完全オリジナル）
- **Echoes of Asterveil（エコーズ・オブ・アスターヴェイル）**
- **星環のリライト（せいかんのリライト）**
- **黎明継承記ルミナクレスト**
- **Fragments of the Ninth Dawn（フラグメンツ・オブ・ザ・ナインス・ドーン）**

### コア体験
1. 王道冒険の高揚感（未知の地域・国家・遺構を巡る）
2. 積み重ねで泣ける仲間ドラマ（離脱・再会・赦し・継承）
3. 直感的だが深いコマンド戦闘（弱点・ブレイク・連携）
4. 本編後も続く成長と挑戦（高難度・厳選・協力レイド）

### 想定ターゲット
- メイン：20〜40代のJRPG好き（ストーリー重視＋戦術好き）
- サブ：スマホ中心だが骨太RPGを求める層
- ライト〜ヘビー混在を想定（難易度選択と育成導線で吸収）

### 差別化ポイント
- **「記憶改編」テーマを戦闘・育成・シナリオに一貫実装**
- ストーリーはソロ完結、マルチはエンド補助で分離
- PC/スマホ共通データ設計で運営効率化
- “やや難しめ”でも初見殺しを避ける情報提示（予兆UI）

### プレイ時間目安
- メインシナリオ：**60〜80時間**
- サブ含む標準クリア：**90〜120時間**
- やり込み＋エンドコンテンツ：**200〜400時間**

## 2. ゲーム全体コンセプト

| 項目 | 提案 |
|---|---|
| ジャンル | 長編ストーリー型・ターン制コマンドRPG（シングル主軸＋協力補助） |
| 提供価値 | 「泣ける長編」と「戦略育成」の両立 |
| セッション設計 | 5〜20分の短時間プレイ（素材周回）と2〜3時間の物語進行を両立 |
| 想定価格軸 | Steam買い切り + モバイル買い切り/ハイブリッド |

### プレイヤー体験の中心
- 章終盤ごとに感情ピークを置く「連続ドラマ構造」
- 戦闘の学習→実践→突破を短サイクルで提供
- 育成は「選択の違いがプレイ体感に出る」設計

### 作品の魅力の柱（5本）
1. **継承と赦しの群像劇**
2. **弱点制御＋ブレイク連携の高戦略バトル**
3. **役割の明確な仲間編成とビルド多様性**
4. **物語価値を持つサブクエスト設計**
5. **ソロ完結を壊さない協力エンド設計**

### 成立性（市場・運営）
- JRPG需要は「重厚シナリオ」「周回性」「配信映え」が強い
- マルチを本編必須にしないことで開発リスクを抑制
- データ駆動イベントで継続運営コストを圧縮可能

### 市場ポジション
- AAA超大型より下、インディー小規模より上の**AA+帯**
- 強み：長編脚本 + 手堅い戦闘 + 運営拡張性

### 収益モデル比較
| モデル | メリット | デメリット | 本企画との相性 |
|---|---|---|---|
| 買い切り | ブランド形成しやすい | 継続売上が弱い | ◎（Steam向け主軸） |
| 基本無料 | LTV伸ばせる | 設計複雑・倫理配慮必要 | △（本編体験を損ねやすい） |
| 買い切り＋DLC | 本編品質と運営両立 | DLC制作体制が必要 | **最適** |

## 3. 世界観

### 舞台設定
世界名：**アスターヴェイル**。天体観測文明が発展した多層世界。空に浮かぶ「記憶環（メモリア・リング）」が各地の歴史情報を蓄積している。

### 時代感
- 産業革命前後＋魔導工学
- 帆船・機関車・携帯端末的な「記録晶片」が共存

### 世界構造
- 中央大陸（5国家）
- 外縁群島（少数民族文化圏）
- 地下遺構帯（古代機構）
- 上層空域（失われた観測都市）

### 文明レベル
- 日常は中世〜近世相当
- 軍事・医療・通信の一部に高魔導技術

### 国家・勢力・宗教・組織
- **燈盟連邦**：交易と工学の国家連合
- **白礫教庁**：記憶環を神託とみなす宗教勢力
- **境界監査院**：遺構管理と封印権限を持つ超法規機関
- **灰冠団**：記憶改竄技術を闇流通する組織

### 種族・文化圏
- 人間、ノクシア（夜目種）、巨躯種ヴァルド、羽民エイル
- 混血差別や同化政策など、章ごとの社会課題に連動

### 世界規模の危機
「**第九黎明現象**」：過去改編の反動で現在の因果が崩壊、都市が“存在履歴”を失って消える。

### 主人公の旅立ち理由
主人公の故郷が存在履歴を失い、家族の記録だけが不自然に欠落。真相を追うため旅立つ。

### ストーリーテーマ
- 喪失と継承
- 真実よりも「選び直した責任」
- 他者を赦すことは自分を赦すこと

### 感動の核
- 仲間同士の相互継承（技・信念・役割）
- 終盤で「救済＝犠牲」ではない第三の選択を成立させる積み上げ

- **転（8〜10章）**：味方陣営の裏切り、主人公の起源発覚
- **結（11〜12章）**：世界再編をめぐる最終選択と継承

- 第7章で精神的支柱キャラが死亡
- 第8章で主人公の出生／記憶が改編産物と判明

### 終盤のどんでん返し
- 黒幕は破壊者でなく「崩壊回避の管理者」
- 主人公側の勝利条件を単純善悪から再定義

### ラスト感動ポイント
- 仲間の専用行動で“主人公だけが犠牲”ルートを覆す
- これまでの個別クエスト完了数で救済演出が増える

### エンディング後の余韻・拡張余地
- 各国復興と新秩序の短編群像
- 消えた観測都市への航路が開く（大型DLC導線）

## 6. バトル仕様

### 基本ループ
1. 敵情報確認（弱点・予兆）
2. 行動選択（攻撃/スキル/防御/道具/連携）
3. ブレイク値蓄積
4. ブレイク中に高火力連携
5. リソース回復ターン管理

### 行動順
- **SPD＋行動重量（Weight）**でCTB順表示
- 重スキルは次行動が遅れる
- 遅延・加速で順番を操作可能

### 戦闘要素定義
- 通常攻撃：SP回収＋軽ブレイク
- スキル：SP消費、役割特化
- 必殺技（アストラ）：ゲージ式、ブレイク中追加効果
- 属性：炎/氷/雷/風/光/影/時/無
- 状態異常：毒、裂傷、沈黙、拘束、鈍化、脆弱
- バフ/デバフ：攻防速・命中・耐性・ヘイト
- 弱点：属性＋武器種（斬/打/射/術）

### 戦術コア
- **ブレイク**：被弾時に被ダメ増・行動遅延
- **スタン**：特定条件で1行動停止
- **連携**：条件達成で追撃（例：支援→狙撃→斬撃）
- **領域効果**：地形/天候が戦況変化

### 役割差
- Vanguard（盾）
- Striker（主火力）
- Tactician（行動操作）
- Healer/Cleanser
- Disruptor（妨害）

### 難しめにする理由と対策
- 理由：選択価値を高めるため
- 対策：
  - 予兆表示（次ターン大技）
  - 敵図鑑で学習補助
  - 難易度「Standard / Veteran / Story」

### 進行別戦略変化
- 序盤：弱点把握＋回復管理
- 中盤：連携と行動順制御
- 終盤：領域管理と複合ギミック対応

### 雑魚戦テンポ
- 2〜4ターン決着を目標
- 「簡易作戦オート」可（危険技時は手動介入推奨通知）

### ボス戦思想
- 3フェーズ制（学習→対応→逆転）
- 失敗理由が可視化される設計

### オート/倍速
- オート：周回向け限定ON（高難度は半自動）
- 倍速：x1/x2/x3（演出スキップ別設定）

### スマホ操作快適性
- 右親指主操作UI＋長押し詳細
- コマンド階層2段まで
- 行動順タイムライン固定表示

### マルチ戦闘分担
- 2〜4人で1キャラずつ持ち込み
- 役割推奨タグ（盾/支援/火力）
- 野良は自動不足ロール補正バフ

## 7. 成長・育成仕様

### レベル・ステータス
- Lv1〜80（本編）、80〜120（エンド）
- STR/INT/VIT/AGI/DEX/RES + BRK効率

### スキル習得
- レベル習得 + **星紋ボード**（分岐式）
- ボードは3系統：攻勢/制御/継戦

### 固有育成
- 各キャラに「誓約特性」2択（例：回復特化 or 支援特化）
- 再振り直し素材あり（課金非必須）

### 装備
- 武器/防具/アクセ2枠/紋章1枠
- レアリティ：C/U/R/SR/UR/AR（ARはエンド中心）

### 厳選とビルド多様性
- ランダムOPは**範囲固定＋天井再抽選**で沼対策
- セット効果は2/4部位、過剰最適解を抑制

### パーティ全体育成
- 共通研究ツリー（倉庫拡張、素材効率、状態異常耐性）
- 控えメンバーに経験値共有率70%

### 育成詰まり対策
- 章進行に応じた推奨育成レベルを提示
- 不足素材の逆引き導線
- 週次で救済パック（ゲーム内通貨）

### ライト/ヘビー両立
- ライト：推奨装備自動提案
- ヘビー：詳細シミュレーション、ログ比較

### インフレ対策
- 係数上昇よりギミック追加で拡張
- 新装備は“横強化”比率を高める
- 旧装備救済の覚醒ルート提供

### 長期拡張性
- ボード枝追加、役割シナジー追加が容易なデータ構造

## 10. フレンド / マルチ機能仕様

### 基本設計
- 本編：完全ソロ完結
- マルチ：素材効率/高難度攻略の補助

### 仕様要点
- フレンド登録：ID・QR・プラットフォーム横断検索
- サポート貸出：1日ごと更新、使用回数で友情pt
- 協力バトル：2〜4人、ルーム/自動マッチ両対応
- コミュニケーション：スタンプ・定型文（暴言対策）
- ギルド：必須ではなく「任意の小規模同盟（最大30）」
- 継続接点：週次協力ミッション、協力ログ称賛

### 報酬方針
- マルチ限定は**時短系・外装系中心**
- 戦力差が出る報酬はソロ入手経路を必ず用意

### 通信同期
- 基本はサーバー権威（Authoritative）
- ターン確定時のみ厳密同期、演出はクライアント補間

### 不正対策
- ダメージ再計算をサーバー側で検証
- 速度改変検知、リプレイ監査、BAN段階制

### コスト最適化
- 常時対戦ではなく「協力インスタンス型」でサーバー負荷を抑える

## 11. プラットフォーム戦略（Steam / iOS / Android）

| 項目 | Steam | iOS/Android |
|---|---|---|
| 操作 | キーボード/パッド最適 | タップ＋ジェスチャ |
| UI情報量 | 多め可 | 2層分割・フォント拡大 |
| セーブ | ローカル＋クラウド | 端末＋アカウント同期 |
| 価格 | 買い切り + DLC | 買い切り or ハイブリッド |

### クロスセーブ
- 可能（アカウント連携）
- 課金通貨はプラットフォーム規約準拠で分離管理

### 販売戦略
- Steam：本編価値訴求、体験版、レビュー導線
- モバイル：序盤無料体験 + 本編アンロック（買い切り寄り）

### リリース順推奨
1. Steam先行（品質評価・バランス取得）
2. 3〜6か月後モバイル最適版
3. クロスセーブ解禁

## 12. 技術選定と開発要件

### 推奨エンジン
- **Unity（2022 LTS以降）**
- 理由：
  - モバイル最適化資産が豊富
  - Addressables/Remote Config運用が成熟
  - ターン制UI実装と運営更新の相性が高い

### 推奨アーキテクチャ
- クライアント：Feature単位のレイヤード（Presentation/Application/Domain/Infra）
- データ：マスターデータ駆動（JSON/ScriptableObject生成）
- サーバー：BFF + Match + Progress API分離

### 分離設計
- シングル戦闘は完全ローカル再現可能
- マルチは専用バトルモジュールを疎結合化

### データ管理
- シナリオ：章/シーン/条件分岐をID管理
- バトル：スキル効果をDSL化（係数・条件・対象）
- イベント：トリガーテーブル＋報酬テーブル

### セーブ設計
- オート3枠＋手動10枠
- 破損対策：世代管理＋CRC
- コンフリクト時は新しい進行度を提案

### 職種と規模目安
- 35〜55名（2年計画）
- 必須：GD, TD, Client, Server, UI/UX, Scenario, Level, TA, QA, LiveOps
- 外注しやすい：背景3D、モブ、ローカライズ、一部UI制作

### 品質管理
- 戦闘ロジックは自動テスト必須
- シナリオ分岐は差分検証ツールで破綻防止
- 端末性能マトリクスでモバイル熱問題を継続監視

## 14. リスクと対策

| リスク | 具体的問題 | 対策 |
|---|---|---|
| 長編シナリオ破綻 | 執筆遅延・整合崩れ | 章ごとロック、先にプロット凍結 |
| コンテンツ過多 | 工数膨張 | Must/Should/Couldで段階実装 |
| マルチ実装コスト増 | 通信バグ多発 | 本編分離、協力コンテンツ限定 |
| スマホ最適化難 | 発熱/電池/視認性 | 30fpsモード、UIスケール可変 |
| 戦闘複雑化 | 離脱率上昇 | チュートリアル分割、予兆UI |
| やり込み単調化 | 周回疲れ | 変則ルールと週替わり目標 |
| エンド寿命短い | 消費速度過多 | シーズン制＋報酬復刻 |
| 運営負荷 | 更新が回らない | データ駆動化、再利用テンプレ |
| ソロ/マルチ格差 | 不公平感 | 戦力報酬はソロ経路保証 |
| PC/スマホ体験差 | 操作不満 | UI別最適化、同期テスト常設 |

## 15. 最終提案まとめ

### この企画の強み
- 感情線とゲーム線の両立設計
- ソロ主軸で開発リスクを制御
- 長期運営へ自然接続できる構造

### 弱み
- 初期開発規模が大きい
- シナリオ品質依存度が高い
- 端末最適化に継続工数が必要

### 絶対に残すべき要素
1. 12章構成の長編本編
2. 弱点×ブレイク×連携の戦闘コア
3. 仲間個別クエストによる感情回収

### 削るなら後回し
- ギルド機能の拡張
- 一部エンドコンテンツ（タイム/スコア）
- 高コスト演出の過剰実装

### 最初に作るべき試作範囲
- 1章丸ごと（探索・会話・ボス）
- 戦闘コア（8属性、状態異常6種、ブレイク）
- 仲間3名の役割差
- スマホUI実地検証

### 企画成立の最重要判断
- **「戦闘が気持ちよく、物語が続きが気になる」この2点が同時に成立するか**
