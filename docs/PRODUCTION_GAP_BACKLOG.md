# PRODUCTION GAP BACKLOG

本書は、製品化ギャップを Must / Should / Could で固定し、既存 `docs/DELIVERY_BACKLOG.md` と整合する実行順を定義する。

## 1. Must（リリース前必須）

### M-1: Steamデモ向け最小クライアント導線整備
- 目的: CLI前提を解消し、一般ユーザーが迷わず開始/終了できる状態を作る。
- 背景: 現状は内部検証向け導線が中心。
- 完了条件: タイトル/開始/設定/終了/再開の基本導線が定義・実装される。
- 依存: app層の既存Playable Slice。
- リスク: UI実装が肥大化し、本来の検証対象がぼやける。

### M-2: セーブ互換運用ルールと移行テスト
- 目的: Early Access以降でセーブ破損事故を防ぐ。
- 背景: `save_data_v1` はあるが移行運用が未固定。
- 完了条件: 互換ポリシー、移行手順、互換テスト観点が文書化される。
- 依存: saveモジュール、save_contract。
- リスク: バージョン更新時の不整合。

### M-3: 配布/ビルド手順の最小確立（Steam向け）
- 目的: 再現可能な提出候補ビルドを作れるようにする。
- 背景: 実装は進んでいるが配布手順が未定義。
- 完了条件: ビルド手順、成果物確認項目、担当境界が文書化される。
- 依存: 開発環境標準化、実行エントリポイント固定。
- リスク: 手順の属人化。

### M-4: QA/デバッグ最小チェックリスト整備
- 目的: Demo公開前の進行不能・クラッシュ検出率を上げる。
- 背景: unittest中心で、プレイ観点の回帰定義が不足。
- 完了条件: 主要導線（開始/戦闘/クエスト/セーブ/復帰）の手動確認項目が定義される。
- 依存: 現行テスト、Playable Slice。
- リスク: 手動確認漏れによる公開事故。

### M-5: ストア公開最低素材タスク分解
- 目的: 開発完了後の申請遅延を防ぐ。
- 背景: ストア説明・画像・審査資料が未分解。
- 完了条件: 必要素材一覧、作成責任、締切が一覧化される。
- 依存: リリーススコープ確定。
- リスク: 実装完了後に公開できない。

## 2. Should（リリース前に高優先で推奨）

### S-1: パッド対応/入力切替の実機回帰
- 目的: Steamユーザーの主要操作環境を担保する。
- 背景: 設計方針はあるが実機基準が不足。
- 完了条件: 主要画面でキーボード/パッド遷移が通る。
- 依存: M-1。
- リスク: 操作不能レビュー。

### S-2: チュートリアル・導線文言の最小ローカライズ方針
- 目的: 新規ユーザーの離脱を抑える。
- 背景: CLI/開発者知識前提の導線が残る。
- 完了条件: 初回30分の学習導線（説明順・文言責務）が定義される。
- 依存: M-1, M-4。
- リスク: 戦闘理解不足。

### S-3: クラッシュ収集/ログ運用の一次対応
- 目的: Early Access移行時の不具合対応速度を上げる。
- 背景: テレメトリ方針に対して運用実装が薄い。
- 完了条件: クラッシュ報告導線、ログ取得手順、トリアージ担当が定義される。
- 依存: M-3。
- リスク: 再現不能バグの長期化。

## 3. Could（リリース後拡張）

### C-1: フレンド協力機能の段階導入
- 目的: コア体験を維持したまま協力要素を追加する。
- 背景: 企画価値は高いが初回必須ではない。
- 完了条件: 限定ルールの試験実装計画が策定される。
- 依存: Early Access運用安定化。
- リスク: 同期不具合で本編評価を毀損。

### C-2: モバイル同時展開準備
- 目的: 将来の市場拡張に備える。
- 背景: 現状はSteam先行が現実的。
- 完了条件: 端末要件、審査要件、UI差分対応の見積が完了。
- 依存: Steam配布基盤。
- リスク: 先行開発の遅延。

### C-3: 長編シナリオ/エンドコンテンツの本格拡張
- 目的: 1.0価値を高める。
- 背景: 現段階は縦切り成立を優先。
- 完了条件: 章追加テンプレートと供給計画が運用可能。
- 依存: コンテンツ量産パイプライン。
- リスク: 開発期間の長期化。

## 4. 既存 feature backlog との整合

- `DELIVERY_BACKLOG.md` の Must Epic（Save/Data, UI/UX, Platform Readiness）を製品化観点で再配置した。
- 本書は「新機能追加」ではなく「リリース可否を決める不足」を優先順位化する補助バックログである。
- 直近PR #54 の工房導線強化は、C-3（コンテンツ拡張）に寄与する一方、M-1〜M-5は未着手領域として維持する。
